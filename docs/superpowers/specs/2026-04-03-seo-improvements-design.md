# SEO Improvements Design

**Date:** 2026-04-03
**Status:** Approved

## Overview

Add comprehensive SEO support to LighterPack using the `@nuxtjs/seo` module suite. Focus on the two public-facing page types: the welcome/landing page and share pages (`/r/[id]`). Authenticated app pages remain SPA and are intentionally excluded from SEO.

## Approach

Use the `@nuxtjs/seo` meta-module, which bundles `@nuxtjs/sitemap`, `@nuxtjs/robots`, `nuxt-og-image`, `nuxt-schema-org`, and `nuxt-link-checker` as a single cohesive package. This is the community standard for Nuxt 3 SEO.

## Module Setup & Global Config

### nuxt.config.ts changes

- Add `'@nuxtjs/seo'` to `modules`
- Add route rule: `'/welcome': { ssr: true }` (welcome page must be SSR for search engine indexing)
- Add `site` config with `url` (from config or env) and `name: 'LighterPack'`
- Add global `app.head` meta: charset, viewport, default description ("Track your gear weight for hiking, backpacking, and adventures")

### Robots.txt (auto-generated by module)

- Allow: `/welcome`, `/r/*`
- Disallow: `/`, `/signin`, `/api/`
- Include sitemap reference

## Sitemap

The `@nuxtjs/sitemap` module auto-discovers routes but needs customization since most routes are behind auth.

### Static routes

- `/welcome` — public landing page, `changefreq: 'monthly'`

### Dynamic routes

- `/r/[id]` — all share pages, `changefreq: 'weekly'`
- Populated via a custom Nitro endpoint (`/api/__sitemap__/urls`) that queries the `lists` table for all rows where `external_id IS NOT NULL` (these are lists that have been shared at least once)

### Excluded routes

- `/` (authenticated app)
- `/signin`
- `/api/**`

## Welcome Page SEO (`app/pages/welcome.vue`)

### Meta tags

- `title`: `'LighterPack — Track Your Gear Weight'`
- `description`: `'LighterPack helps you track the gear you bring on adventures. Enter packing lists, visualize pack weight, and share with the community.'`
- `ogType`: `'website'`
- `ogUrl`: canonical URL for `/welcome`
- `ogImage`: static branded OG image
- `ogDescription`: mirrors description
- `twitterCard`: `'summary_large_image'`
- `twitterTitle`, `twitterDescription`: mirror OG values
- Canonical `<link>` to `/welcome`

### Structured data

- `WebApplication` schema: name, description, URL, applicationCategory: "Utility"

## Share Page SEO (`app/pages/r/[id].vue`)

### Meta tags

- `title`: `"{List Name} - LighterPack"`
- `description`: dynamic — e.g., `"A {N}-item gear list weighing {total weight}. Shared on LighterPack."`
- `ogType`: `'website'`
- `ogUrl`: canonical URL for the share page
- `ogImage`: dynamically generated by `nuxt-og-image`
- `twitterCard`: `'summary_large_image'`
- `twitterTitle`, `twitterDescription`: mirror OG values
- Canonical `<link>` via `useHead`

### Structured data

- `ItemList` schema with `numberOfItems` and `name` from the list data

## Dynamic OG Images

### Module

`nuxt-og-image` using Satori (same engine Vercel uses) to render Vue components as images server-side.

### Default OG image component

`app/components/OgImage/OgImageDefault.vue` — the module's convention for the default template.

### Share page OG image

- Branded card with LighterPack wordmark at top
- List name as the main heading
- Key stats: total weight, item count, category count
- Amber/charcoal color scheme matching the app's design tokens
- Generated on-demand, cached by the module
- Props passed via `defineOgImage` with list name, weight, item count

### Welcome page OG image

- Static branded card: "LighterPack" wordmark, tagline "Every ounce accounted for.", app description
- Same amber/charcoal color palette

### Satori constraints

- Flexbox only (no CSS grid)
- Limited CSS property support
- Component must use inline styles or Satori-compatible subset

## Out of Scope

- No SEO for authenticated pages (SPA behind auth, correct as-is)
- No blog or content marketing infrastructure
- No complex schema.org beyond WebApplication + ItemList
- No custom link-checker config (module defaults are sufficient)
- No analytics or Search Console integration
